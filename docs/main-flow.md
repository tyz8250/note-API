# Issue 31 で学んだこと

main.go に初期化処理をすべて直書きすると、動作はしても起動の流れが読みづらくなる。
そのため、DB接続、テーブル作成、ルーティング登録、サーバー起動をそれぞれ関数に分けた。

これにより main は「起動時に何をしているか」を上から追える形になった。
main は詳細を書く場所ではなく、アプリ起動の流れを表現する場所だと理解した。

## 詰まったこと

文法の意味がわからなかった。

```go

if err := ensureSchema(db); err != nil {
    log.Fatal(err)
}
```

「ensureSchema(db) を実行して、その結果のエラーを err に入れ、もし err が nil ではなければ log.Fatal(err) を実行する」
